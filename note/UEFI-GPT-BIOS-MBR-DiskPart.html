我们所谓的装系统，其实是：将 Windows 部署到 硬盘驱动器分区类型 与 计算机固件接口 相匹配的设备上。

硬盘驱动器分区类型必须与计算机的固件相匹配，可以将 Windows 安装在基于以下任一固件类型的硬盘驱动器上：

  • 基本输入/输出系统 (BIOS)：使用主引导记录 (MBR) 分区结构。

  • 可扩展固件接口 (EFI)（第 1 类）：使用 GUID 分区表 (GPT) 分区结构。

  • 统一可扩展固件接口 (UEFI) 第 2 类：使用 GPT 分区结构。还包括允许使用 BIOS 功能（包括 MBR 分区结构）的兼容性支持模块 (CSM)。可以在固件中启用或禁用该模块。

  • 统一可扩展固件接口 (UEFI) 第 3 类：使用 GPT 分区结构。

理论部分

1.驱动器(磁盘/硬盘)

  • 硬盘驱动器 (HDD)

  • 固态驱动器 (SSD)

  • 其他驱动器

2.分区类型

  • MBR(Master Boot Record，主引导记录)

    MBR 磁盘分区支持最大卷为 2 TB（Terabytes）并且每个磁盘最多有 4 个主分区（或 3 个主分区，1 个扩展分区和无限制的逻辑驱动器）。

  • GPT(GUID Partition Table，全局唯一标识磁盘分区表 (GUID: Globally Unique Identifier))

    GPT 磁盘分区样式支持最大卷为 128 EB（Exabytes）并且每磁盘的分区数没有上限，只受到操作系统限制。

    GPT 是一个实体硬盘的分区表的结构布局的标准，它是可扩展固件接口（EFI）标准（被Intel用于替代个人计算机的BIOS）的一部分，被用于替代 BIOS 系统中的引导记录（MBR）分区表。

    UEFI/GPT 替代 BIOS/MBR。

3.固件类型

在开机到引导操作系统期间，由 BIOS 固件控制系统。

固件接口规范：

  • Legacy BIOS (Basic Input/Output System，基本输入输出系统)，传统固件接口。

    开机 -> BIOS 初始化 -> BIOS 自检 -> 引导操作系统 -> 进入系统

    BIOS 计算机主板上一个 ROM 芯片上的程序，保存着计算机最重要的基本输入输出的程序、开机后自检程序(Power On Self Test)和系统自启动程序，它可从CMOS中读写系统设置的具体信息。初始化硬件，检测硬件，为计算机提供最底层的、最直接的硬件设置和控制。

  • UEFI (Unified Extensible Firmware Interface，统一可扩展固件接口)，全新类型接口标准。

    开机 -> UEFI 初始化 -> 引导操作系统 -> 进入系统

    对比Legacy，UEFI支持大于 2 TB 的硬盘驱动器分区， 支持一个驱动器上的分区超过 4 个， 快速引导， 高效的电源和系统管理， 强大的可靠性和故障管理，

4.部署 Windows， 驱动器的分区规则

  • 在将 Windows 部署到基于 BIOS 的设备时，必须使用 MBR 文件系统格式化硬盘驱动器。Windows 在基于 BIOS 的计算机上不支持 GUID 分区表 (GPT) 文件系统。

  • 当将 Windows 部署到基于 UEFI 的设备时，必须使用 GUID 分区表 (GPT) 文件系统对包含 Windows 分区的硬盘驱动器进行格式化。其他驱动器可以使用 GPT 或主启动记录 (MBR) 文件格式。

操作部分

选择固件模式

启动到 UEFI 模式或旧版 BIOS 模式：

  1. 启动电脑，按 Esc、F2(适用于大部分主板)、F9、F12 或其他键，进入固件或启动菜单(BIOS设置，新主板要进入高级BIOS设置(Advanced BIOS Features))。

  2. 从 BIOS Main 菜单屏幕中，选择 Boot。

  3. 从 Boot 屏幕中，选择 UEFI/BIOS Boot Mode，然后按 Enter 键。

    此时将显示 UEFI/BIOS Boot Mode 对话框屏幕。

  4. 使用向上和向下方向键选择相应的 Legacy 或 UEFI 引导模式，然后按 Enter 键。

1.1 设置 UEFI/GPT

  • Boot mode 项设为 UEFI only

  • UEFI 第 2 类，如果要使用 BIOS/MBR 兼容性支持模块(CSM)禁用 UEFI Secure Boot 功能

    • Launch CSM 项设为 Enabled
    • Secure BOOT Control 项设为 Disabled

  • 另外，某些老旧主板，以USB启动盘为例，进行以下操作：

    • 解压 windows.iso 原版安装镜像，到U盘(FAT32格式)的根目录；

    • 解压工具打开(不是解压) 安装镜像中的install.wim:sources\install.wim\1\Windows\Boot\EFI\bootmgfw.efi

    • 拷贝Bootmgfw.efi文件到安装盘\efi\boot\重命名为bootx64.efi

    • 拷贝\efi\microsoft\boot\下所有文件到\efi\boot\

    • 找到 Launch EFI Shell from filesystem device 进入 UEFI Shell 界面

      • 进入第 N(0~n) 个磁盘分区

        $ fs0

      • 列出这个磁盘分区的所有文件和文件夹

        $ ls

      • 如果没有看到复制的Bootmgfw.efi和Shell.efi文件，说明fs0不是安装文件所在分区，

      • 就接着查看fs1、fs2、fs3… 直到看到复制的两个安装文件

        $ bootmgfw.efi

      • 进入系统安装

1.2 BIOS/MBR 固件设置

  1. Launch CSM 项设为 Enabled

  2. Secure BOOT Control 项设为 Disabled

整数分区计算方法
NTFS格式，硬盘有255磁头，63扇区，每柱面大小为：
512byte x 255 x 63＝8225280bytes ＝7.84423828125 MB

分100 GB，100×1024 MB=102400 MB
需柱面数为：102400÷7.84423828125=13054.16744475568004979769685652
向上取整数：13055个柱面

应分 MB 数：13055×7.84423828125=102406.5307617188 MB
向上取整数：102407 MB，系统为整数100.00 GB。

Windows 10 自动创建 566 MB 隐藏分区

手动擦除驱动器并将其转换为 GPT

  • 选择 UEFI 固件模式，以DVD光驱或U盘启动计算机

  • 在 Windows 安装程序执行到选择安装磁盘时，按 Shift+F10 打开命令提示符窗口

  • 打开 diskpart 工具:

    $ diskpart

  • 标识要重新格式化的驱动器

    $ list disk

  • 选择该驱动器，然后将其重新格式化

    $ select disk <disk number>

    $ detail disk

    $ clean

    $ convert gpt

    $ exit

  • 关闭命令提示符窗口，继续进行 Windows 安装程序安装

    • 在界面"您想进行何种类型的安装?"时，选择"自定义(高级)"；
    • 驱动器将显示为包含"未分配空间"的单个区域；
    • 选择"未分配的空间"，然后单击"驱动器选项(高级)"；
    • 单击"新建"，按提示创建分区；
    • 选择第一个主分区，"下一步" 开始进行 Windows 安装。

DiskPart 直接在PE里执行

实用工具分区或数据分区，应放置在 Windows 分区之前或 Windows RE 分区之后，

将 Windows 分区和恢复分区放置在一起，当 Windows RE 更新时，Windows 通过缩小 Windows 分区扩大 Windows RE 分区。

Windows 7, Windows Server 2008 R2

  • 基于 BIOS/MBR 的硬盘驱动器分区

    • 系统分区，300 MB NTFS格式 驱动器号(S) *
    • Windows 分区，磁盘剩余空间 NTFS格式 驱动器号(W) *
    • 恢复工具分区，（例如 Windows RE）自定义 Windows 恢复映像的大小

  • 基于 UEFI/GPT 的硬盘驱动器分区

    • EFI 系统分区 (ESP) ，100 MB FAT32格式 驱动器号(无) *
    • Microsoft(R) 保留分区 (MSR)，128 MB 驱动器号(无) *
    • Windows 分区，磁盘剩余空间 *
    • 恢复工具分区，250 MB

具体实现参考下面的 Windows 10

Windows 10

  • 基于 BIOS/MBR 的硬盘驱动器分区

    • 系统分区，100 MB NTFS格式 驱动器号(S) *
    • Windows 分区，磁盘剩余空间 NTFS格式 驱动器号(W) *
    • 恢复工具分区（Windows RE），350 MB *
    • 数据分区
    • 默认分区布局为：系统分区、Windows 分区和恢复工具分区
    rem == CreatePartitions-BIOS.txt ==
    rem == These commands are used with DiskPart to
    rem    create three partitions
    rem    for a BIOS/MBR-based computer.
    rem    Adjust the partition sizes to fill the drive
    rem    as necessary. ==
    select disk 0
    clean
    rem == 1. System partition ======================
    create partition primary size=100
    format quick fs=ntfs label="System"
    assign letter="S"
    active
    rem == 2. Windows partition =====================
    rem ==    a. Create the Windows partition =======
    create partition primary
    rem ==    b. Create space for the recovery tools
    shrink minimum=500
    rem       ** NOTE: Update this size to match the
    rem                size of the recovery tools
    rem                (winre.wim)                 **
    rem ==    c. Prepare the Windows partition ======
    format quick fs=ntfs label="Windows"
    assign letter="W"
    rem == 3. Recovery tools partition ==============
    create partition primary
    format quick fs=ntfs label="Recovery"
    assign letter="R"
    set id=27
    list volume
    exit
    rem DiskPart /s F:\CreatePartitions-BIOS.txt

  • 基于 UEFI/GPT 的硬盘驱动器分区

    • EFI 系统分区 (ESP，100 MB FAT32格式 驱动器号(无) ；对于 Advanced Format 4K Native (4-KB-per-sector) 驱动器，由于 FAT32 文件格式的限制，大小最小为 260 MB。 *
    • Microsoft(R) 保留分区 (MSR)，16 MB 驱动器号(无) *
    • 其他实用程序分区
    • Windows 分区，磁盘剩余空间 *
    • 恢复工具分区（Windows RE），350 MB ，微软建议应至少具有 1 GB 的可用空间，Type ID: DE94BBA4-06D1-4D40-A16A-BFD50179D6AC*
    • 数据分区
    • 默认分区布局为：系统分区、MSR、Windows分区和恢复工具分区
    rem == CreatePartitions-UEFI.txt ==
    rem == These commands are used with DiskPart to
    rem    create four partitions
    rem    for a UEFI/GPT-based PC.
    rem    Adjust the partition sizes to fill the drive
    rem    as necessary. ==
    select disk 0
    clean
    convert gpt
    rem == 1. System partition =========================
    create partition efi size=100
    rem    ** NOTE: For Advanced Format 4Kn drives,
    rem               change this value to size = 260 **
    format quick fs=fat32 label="System"
    assign letter="S"
    rem == 2. Microsoft Reserved (MSR) partition =======
    create partition msr size=16
    rem == 3. Windows partition ========================
    rem ==    a. Create the Windows partition ==========
    create partition primary
    rem ==    b. Create space for the recovery tools ===
    shrink minimum=500
    rem       ** NOTE: Update this size to match the
    rem                size of the recovery tools
    rem                (winre.wim) plus free space                   **
    rem ==    c. Prepare the Windows partition =========
    format quick fs=ntfs label="Windows"
    assign letter="W"
    rem === 4. Recovery tools partition ================
    create partition primary
    format quick fs=ntfs label="Recovery tools"
    assign letter="R"
    set id="de94bba4-06d1-4d40-a16a-bfd50179d6ac"
    gpt attributes=0x8000000000000001
    list volume
    exit
    rem DiskPart /s F:\CreatePartitions-UEFI.txt
