Windows 设置

Windows 共享设置

1.Windows+R（运行命令）：secpol.msc

  • 本地策略 - 用户权限分配
    • 拒绝本地登陆：删除其中的 Guest
    • 拒绝从网络访问这台计算机：删除其中的 Guest
  • 本地策略 - 安全选项
    • 网络访问：本地账户的共享和安全模式：仅来宾-对本地用户进行身份验证，其身份为来宾

2.启用 Guest 账户

  • Windows 10. 运行命令：compmgmt.msc
    计算机管理 > 系统工具 > 本地用户和组 > 用户 > 右侧 Guest 鼠标右键 > 属性 > 常规 > 取消勾选“账户已禁用 > 确定
  • Windows 7. 运行命令：control userpasswords
    管理其他账户 > 启用 Guest
  • 方法二. 运行命令：secpol.msc
    本地策略 > 安全选项 > 账户：来宾账户状态：已启用 > 确定

3.要共享的文件夹

  • 鼠标右键：属性 > 安全 > 编辑(E) > 添加(D) > 高级(A)... > 立即查找(N) > Guest > 确定
  • 鼠标右键：共享 > 下拉框选择 Guest > 添加(A)

4.上面操作还不能共享

  • 尝试运行命令：Control firewall.cpl
  • 左侧：允许程序或功能通过 Windows 防火墙 > 勾选：文件和打印机共享

5.查看计算机共享文件

  • 运行命令：compmgmt.msc
    计算机管理 > 系统工具 > 共享文件夹 > 共享

6.关闭密码保护共享

  • 运行命令：control.exe /name Microsoft.NetworkAndSharingCenter
    控制面板 > 网络和 Internet > 网络和共享中心 > 更改高级共享设置
    所有网络：密码保护的共享 > 关闭密码保护共享

Windows 设置

关闭家庭组

services.msc / 控制面板 > 管理工具 > 服务 > HomeGroup Listener 和 HomeGroup Provider 禁用

WindowsDefender

  1. 控制面板 > Windows Defender > 设置 > 实施保护 去掉勾
  2. 服务 WindowsDefenderService 禁用

关闭磁盘碎片整理计划

选中磁盘 > 属性 > 工具 > 对驱动器进行优化和碎片整理 > 优化 > 更改设置 > 取消选择按计划运行

Superfetch 服务

服务 > Superfetch > 启动类型 > 自动（延迟启动）

清理产生的 Windows.old 文件夹

  1. rd c:windows.old /s/q
  2. C 盘 > 右键 > 属性 > 磁盘清理 > 选中"以前的 Windows 安装"复选框 > 确定清理

设置 Windows 自动登陆

  1. netplwiz,取消使用计算机必须输入用户名和密码的选项
  2. 双击需要自动登录的账户，输入密码

关闭 ipv6

网络共享中心 > 网络连接 > 以太网 > 属性 > 取消 ipv6

开启 HybridBoot

  1. powercfg.cpl/控制面板 > 电源选项 > 左侧：选择电源按钮的功能 > 更改当前不可用的设置 > 关机设置 > 勾上启用快速启动
  2. 开启休眠：powercfg–h on，关闭休眠：powercfg –h off

关闭性能特效

sysdm.cpl/系统属性 > 高级 > 性能 > 设置 > 关闭淡出淡入效果

第三方软件服务项

msconfig/系统配置 > 服务 > 勾选：隐藏所有的 Microsoft 服务 > 去掉不需要运行的服务

关闭计划任务里的隐藏的自启动程序

taskschd.msc/任务计划程序库 > 禁用右侧任务列表不需要的任务

关闭 windows 安全中心服务

services.msc/服务 > Security Center > 禁用

开机启动

  • %USERPROFILE%\AppData\Roaming\Microsoft\Windows\Start Menu\Programs\Startup 复制快捷方式进去；或：
  • HKEY_LOCAL_MACHINE\SOFTWARE\Microsoft\Windows\CurrentVersion\Run → 新建 → 字符串值，添加可执行文件路径；或：
  • taskschd.msc 任务计划程序 → 创建基本任务

Windows 10 家庭版开启组策略

@echo off
pushd "%~dp0"

dir /b %SystemRoot%\servicing\Packages\Microsoft-Windows-GroupPolicy-ClientExtensions-Package~3*.mum >List.txt
dir /b %SystemRoot%\servicing\Packages\Microsoft-Windows-GroupPolicy-ClientTools-Package~3*.mum >>List.txt

for /f %%i in ('findstr /i . List.txt 2^>nul') do dism /online /norestart /add-package:"%SystemRoot%\servicing\Packages\%%i"
pause

Windows 鼠标右键没有“新建”“新建文件夹”选项

方法一，注册表修改

  • Windows+R : 运行regedit
  • 注册表树结点 : HKEY_CLASSES_ROOT\Directory\Background\shellex\ContextMenuHandlers\
  • 左侧树ContextMenuHandlers上右键“新建(N)”-"项(K)"，命名为“New”
  • 点击New，在右侧面板双击**(默认)**
  • 弹出“编辑字符串”对话框，“数值数据(V)”设值为**{D969A300-E7FF-11d0-A93B-00A0C90F2719}**

方法二，运行 reg 文件

  • 如果没有“新建”
  • Windows+R : 运行notepad
  • 保存下面代码到文本文件，然后重命名为 Restore-Missing-New.reg 运行
Windows Registry Editor Version 5.00
[HKEY_CLASSES_ROOT\Directory\Background\shellex\ContextMenuHandlers\New]
@="{D969A300-E7FF-11d0-A93B-00A0C90F2719}"

  • Windows+R : 运行notepad
  • 保存下面代码到文本文件，然后重命名为 Restore-Missing-New-Folder.reg 运行

Firefox

Windwos+R:firefox -ProfileManager

创建配置文件 - 输入新的配置文件名称，然后一定要选择文件夹，新建文件夹，命名后选择这个新文件夹，不然是随机名 - 完成。

一定要去掉启动时不询问并使用选定的配置文件勾选，再退出，因为要把一个默认配置的默认启动的给其他人用。

复制一份 Firefox 的快捷方式文件，右键属性 - 快捷方式 - 目标 - 加参数--no-remote -p 配置文件名：

完整启动方式为："C:\Program Files\Mozilla Firefox\firefox.exe" --no-remote -p 配置文件名

这样就得到一个属于自已配置的 Firefox，把这个快捷方式文件命名为iff之类简短的名字然后放在C:\Windows\System32目录，下次直接Windwos+R:iff就启动这个配置的 Firefox 了。

Chrome

同样复制一个新的 Chrome 快捷方式文件命名为igc，属性目标中加--user-data-dir="%appdata%\Google\Chrome\配置目录名"，这样得到一个独立的 Google Chrome 实例。

完整启动方式为："C:\Program Files (x86)\Google\Chrome\Application\chrome.exe" --user-data-dir="%appdata%\Google\Chrome\配置目录名"

Snipaste 截图贴图需要配置："C:\Program Files (x86)\Google\Chrome\Application\chrome.exe" --force-renderer-accessibility --user-data-dir="%appdata%\Google\Chrome\test

热点与电源

Windows 开启热点，自动 Off。

在 devmgmt.msc Device Manager 中的 Network adapters 树结点对 Wireless、Direct Virtual 等子项进行右键属性设置 “Power Management”；

在 regedit 地址栏输入 HKLM\System\ControlSet001\Services\ICSSVC\Settings 右侧面板新建 DWORD (32-bit) Value 类型的 PeerlessTimeout 项，双击/编辑值为 120，以配置没有链接时 Timeout 时间。

浏览器主页被篡改

在 regedit 注册表查看：

# 复制到 Registry Editor 地址栏，然后 Enter

# 1. HKEY_CURRENT_USER
Computer\HKEY_LOCAL_MACHINE\SOFTWARE\Microsoft\Internet Explorer\Main

# 2. HKEY_LOCAL_MACHINE
HKEY_CURRENT_USER\Software\Microsoft\Internet Explorer\Main\Start Page

这两个目录下的 Start Page 值；尝试改回正常的或 new:tabs 空白页；

在火绒中建立自定义规则，监视上述注册表中两个地方的 Start Page 是哪个程序在篡改：

  1. Protection → Advanced 中开启 Custom Rules

  2. Preferences → Advanced → Custom Rules → Add → 切到 Registry rules 选项卡 → 按注册表路径找 Start Page → Action 中勾选 Modify → Save.
