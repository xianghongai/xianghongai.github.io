GnuPG

GnuPG 是一个开放源程序的与OpenPGP 标准兼容的非对称信息加密系统。

概要

GnuPG 由四类组成

  • 私钥文件
  • 公钥文件
  • 公钥回收证书
  • 私钥密码

私钥和公钥运用

文件---->[发件人用私钥数字签名]---->签名文件---->[收件人用发件人公钥验证签名]---->核对

明文---->[发件人用私钥加密]---->密文---->[收件人用发件人公钥解密]---->明文

明文---->[发件人用收件人公钥加密]---->密文---->[收件人用私钥解密]---->明文

签认公钥

  • 有可能伪造公钥信息 (假公章)，需要对公钥进行有效确认
  • 获取对方公钥之后，为了安全起见，务必核对公钥指纹值 (fingerprint)，是否真的为他所拥有
  • 导入公钥、核对公钥指纹值、进行信任操作、用这把绝对确信的公钥进行数据通信

语法

Syntax: gpg [options] [commands] [files]

Options:

 -a, --armor                 以 ASCII 文本形式，而不是二进制格式
 -r, --recipient             加密给收件人
 -u, --local-user            用以签名或解密的用户
 -o, --output                导出为文件

Commands:

-s, --sign                  签名
-b, --detach-sign           分离式签名
-e, --encrypt               加密数据
-c, --symmetric             使用对称密码加密
-d, --decrypt               解密数据（默认）
    --verify                验证签名
-k, --list-keys             列出钥匙
-K, --list-secret-keys      列出私钥

密钥

创建密钥

新建密钥（密钥，包括一把解密用私钥和一把配对的加密用公钥）

$ gpg --gen-key

$ gpg --full-generate-key

导出密钥

# 直接显示
# 公钥
$ gpg -a/--armor --export <USER_ID>
# 私钥
$ gpg -a/--armor --export-secret-keys <USER_ID>

# 导出至文件
# 导出公钥文件，发给别人
$ gpg -a/--armor -o/--output <公钥文件>.key --export <USER_ID>
# 导出私钥文件，不能泄露出去，仅自己用
$ gpg -a/--armor -o/--output <私钥文件>.key --export-secret-keys <USER_ID>

# 上传公钥到公共服务器上
$ gpg --keyserver hkp://subkeys.pgp.net --send-keys <USER_ID>

导入密钥

收到一把别人的公钥，加进钥匙数据库

# 导入钥匙文件
$ gpg --import <公钥或私钥文件>

# 从 Key 服务器导入
$ gpg --keyserver hkp://subkeys.pgp.net --search-keys <USER_ID>
$ gpg --keyserver hkp://subkeys.pgp.net --recv-keys <USER_ID>

# 更新密钥
$ gpg --refresh-keys

核对公钥指纹值

$ gpg --fingerprint <USER_ID>

$ gpg --with-fingerprint <公钥文件>

密钥列表

# 列出所有公钥
$ gpg -k
$ gpg --list-keys

# 列出所有私钥
$ gpg -K
$ gpg --list-secret-keys

$ gpg --list-secret-keys --keyid-format LONG

# 列出签认的密钥
$ gpg --list-sigs

# 列出密钥和指纹
$ gpg --fingerprint

公钥签认

$ gpg --sign-key <USER_ID>
$ gpg --lsign-key <USER_ID>
# 列出并检查密钥签名
$ gpg --check-signatures <USER_ID>

编辑密钥

可以修改钥匙的失效日期、对钥匙签名、添加信任等操作

$ gpg --edit-key <USER_ID>

删除密钥

删除公钥：$ gpg --delete-key <USER_ID>

删除私钥：$ gpg --delete-secret-key <USER_ID>

公钥回收证书

忘记私钥口令、私钥丢失、私钥被盗窃，发布这个回收证书，来声明钥匙不再有效。

# 生成回收证书
$ gpg -o/--output revoke.asc --gen-revoke <USER_ID>

# 告诉其他人这把钥匙作废
# 1. 导入回收证书
$ gpg --import revoke.asc
# 2. 发送回收证书到服务器，声明 Key 作废
$ gpg --keyserver <SERVER_NAME> --send <USER_ID>

加密解密

加密

gpg [options] -e/--encrypt <要加密的文件>

$ gpg -r/--recipient "Receiver" -e/--encrypt <要加密的文件>

$ gpg -a/--armor -u/--local-user "Sender" -r/--recipient "Receiver" -s/--sign -e/--encrypt <要签名和加密的文件>

解密

gpg [options] -d/--decrypt <要解密的文件.gpg>

# 显示解密内容
$ gpg -a/--armor -d/--decrypt <要解密的文件>

# 生成解密文件
$ gpg -o/--output 输出文件名 -d/--decrypt <要解密的文件>

对称加密与解密

加密：$ gpg -c/--symmetric <要加密的文件> 会提示输出临时用的对称密钥/密码，不是私钥密码！

解密：$ gpg -d/--decrypt <要解密的文件.gpg>

签名与验证

为了保证文件传输的完整性、文件来源的可靠性，防止恶意篡改、欺诈钓鱼，对数据进行签名与验证；

发件人 用自己的 私钥 对文件数字签名，再用 收件人 的 公钥 对文件加密；

收件人 用自己的 私钥 对加密文件解密，再用 发件人 的 公钥 对私密文件验证签名。

私钥签名

# 包含原文件数据签名
# 二进制格式
$ gpg -s/--sign <要签名的文件>
# 文本格式，不压缩，文本内容包裹在 ASCII 签名中
$ gpg --clear-sign <要签名的文件>

# 分离式签名
# 生成单独的签名文件，与文件内容分开
# 二进制储存
$ gpg -b/--detach-sign <要签名的文件>
# ASCII 码
$ gpg -a/--armor -b/--detach-sign <要签名的文件>

# 签名&加密
gpg -r/--recipient <收件人> -se/--sign --encrypt <要签名加密的文件>

公钥验证

# 验证签名
$ gpg --verify 签名的文件

# 验证签名&加密
$ gpg --decrypt 签名&加密的文件 > /dev/null
$ gpg --decrypt 签名&加密的文件 > nul

Syntax: gpg [options] [files]
Sign, check, encrypt or decrypt

指令：

 -s, --sign                  make a signature
     --clear-sign            make a clear text signature
 -b, --detach-sign           生成一份分离的签名
 -e, --encrypt               加密数据
 -c, --symmetric             仅使用对称加密
 -d, --decrypt               解密数据(默认)
     --verify                验证签名
 -k, --list-keys             列出密钥
     --list-signatures       列出密钥和签名
     --check-signatures      列出并检查密钥签名
     --fingerprint           列出密钥和指纹
 -K, --list-secret-keys      列出私钥
     --generate-key          生成一副新的密钥对
     --quick-generate-key    quickly generate a new key pair
     --quick-add-uid         quickly add a new USER_ID
     --quick-revoke-uid      quickly revoke a USER_ID
     --quick-set-expire      quickly set a new expiration date
     --full-generate-key     full featured key pair generation
     --generate-revocation   生成一份吊销证书
     --delete-keys           从公钥钥匙环里删除密钥
     --delete-secret-keys    从私钥钥匙环里删除密钥
     --quick-sign-key        quickly sign a key
     --quick-lsign-key       quickly sign a key locally
     --sign-key              为某把密钥添加签名
     --lsign-key             为某把密钥添加本地签名
     --edit-key              编辑某把密钥或为其添加签名
     --change-passphrase     change a passphrase
     --export                导出公钥
     --export-secret-keys    导出私钥
     --send-keys             把密钥导出到某个公钥服务器上
     --receive-keys          从公钥服务器上导入密钥
     --search-keys           在公钥服务器上搜寻密钥
     --refresh-keys          从公钥服务器更新所有的本地密钥
     --import                导入/合并密钥
     --card-status           打印卡状态
     --edit-card             更改卡上的数据
     --change-pin            更改卡的 PIN
     --update-trustdb        更新信任度数据库
     --print-md              print message digests
     --server                以服务器的方式运行
     --tofu-policy VALUE     set the TOFU policy for a key

选项：

 -a, --armor                 输出经 ASCII 封装
 -r, --recipient USER_ID     encrypt for USER_ID
 -u, --local-user USER_ID    use USER_ID to sign or decrypt
 -z N                        set compress level to N (0 disables)

 -o, --output FILE           write output to FILE

 -n, --dry-run               不做任何改变
 -i, --interactive           覆盖前先询问
