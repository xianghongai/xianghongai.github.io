Web 开发环境搭建

安装 NodeJS

  1. 采用 Node Version Manager nvm 安装 Node，采用 nrm 选择 npm 源，安装 Yarn

安装路径 C:\nodejs

# 查看最新 LTS 版本
$ nvm list available
# 安装和使用一个版本
$ nvm install <version>
$ nvm use <version>

检查：$ node -v ,$ npm -v, $ yarn -v

  1. 配置 npm 全局模块存放路径、cache 的路径
$ cd \pro*\nod*
$ md node_global node_cache
$ npm config set prefix "C:\nodejs\node_global"
$ npm config set cache "C:\nodejs\node_cache"
  1. 设定 Node 环境变量

新建 NODE_PATH: C:\nodejs\node_global\node_modules

Path 追加 C:\nodejs\node_global\

  1. 设定 Yarn 环境变量

%localappdata%\Yarn\bin

%localappdata%\Yarn\Data\global\node_modules\.bin

  1. 视网络情况，更换 npm 源
$ yarn global add nrm
# 看看有哪些 npm 镜像源
$ nrm ls
# 使用一个速度快的
$ nrm use <registry>
  1. 全局安装常用 npm 包 (可选择)

$ yarn global add http-server browser-sync terminal-translate eslint gulp-cli grunt-cli npm-check-updates

项目中配置 .yarnrc 或 .npmrc

.yarnrc

phantomjs_cdnurl "http://cnpmjs.org/downloads"
sass_binary_site "https://npm.taobao.org/mirrors/node-sass/"
registry "https://registry.npm.taobao.org"

.npmrc

phantomjs_cdnurl=http://cnpmjs.org/downloads
sass_binary_site=https://npm.taobao.org/mirrors/node-sass/
registry=https://registry.npm.taobao.org

项目中启动前端工作流

在前端项目 package.json 文件所在目录命令行 (CLI) 执行 yarn install 或 npm install 安装依赖包即可，HG/Git 版本控制工具中添加忽略目录 node_modules。

安装完依赖后，查看 package.json 文件中 scripts 属性，一般执行 npm start 或 npm run dev 启动项目。

使用过程中报错，检查权限和 package.json 配置。

开发工具

  • 字体 Source Code Variable
  • Visual Studio Code
  • Visual Studio IDE Community版
  • WebStorm

先安装 Source Code Variable 字体 Share/Font/Source-Code-Pro.zip，提取码 1024，一个适合编码的字体是比较重要的；

计算机配置较好，想折腾，推荐免费的编辑器 Visual Studio Code 并安装扩展调教，
计算机配置一般，或不想折腾，推荐 IDE，独占所有系统资源；

  • Visual Studio Code，扩展中安装 Settings Sync 同步插件，可在设置中设置"sync.gist": "5f0eb9482b96be16eb6865cbc7728d29"项，然后 Ctrl+Shift+P 中执行Sync: Download Settings同步，视网络情况需要一段时间，建议开全局代理。按插件说明进行配置，可以只从公共分享的配置下载配置。
  • IDE，直接选择免费的 Visual Studio IDE Community版，安装时候选择“单个组件”勾选安装适合 Web 开发的组件。

编辑器设置的同步，只是为了同步你的设置和快捷键，方便你在不同的计算机上快速搭建开发工具。

需要 Github 账号，并在 “Settings → Developer settings → Personal access tokens → Generate new token → 勾选 gist” Generate token。务必复制 Token (只会出现一次) 保存在 Foxmail(QQ) 邮箱的记事本中，并备注是何用途；然后再创建一个 gist 用以同步数据，具体的看同步插件页面，有配图教程。

同步不同编辑器配置，可以在 github 创建多个 token 和 gist；

开启代理安装

特殊网络情况可设定代理 (需要代理服务)

npm config set proxy http://127.0.0.1:1080
npm i node-sass
npm config delete proxy
