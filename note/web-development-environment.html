Web开发环境搭建和工具选择

任何软件都应该按默认路径安装。

安装 NodeJS

  • nvm: Node Version Manager
  • nrm: NPM registry manager

  1. 采用 nvm 安装 NodeJS，采用 nvm 选择 npm 源

  2. 检查：$ node -v ,$ npm -v, $ yarn -v

  3. 配置 npm 全局模块存放路径、cache 的路径

     $ cd \pro*\nod*
     $ md node_global node_cache
     $ npm config set prefix "C:\nodejs\node_global"
     $ npm config set cache "C:\nodejs\node_cache"

  1. 环境变量

    新建{"NODE_PATH":"C:\nodejs\node_global\node_modules"}

    追加{"Path":"C:\nodejs\node_global\"}

  2. 安装常用 npm 包

$ npm install -g http-server browser-sync terminal-translate eslint gulp-cli grunt-cli

  1. 安装 Yarn

安装如报错，开全局代理。

npm config set proxy http://127.0.0.1:1080
npm i node-sass
npm config delete proxy

安装 Python

安装 Python 2.7 到 C:\Python27 并配置环境变量

  • 新建{"PYTHON_HOME":"C:\Python27"}
  • 追加{"Path":"%PYTHON_HOME%;%PYTHON_HOME%\Scripts;"}

JDK

  1. 默认路径安装，配置环境变量
  • 新建{"JAVA_HOME":"JDK安装路径"}
  • 追加{"Path":";%JAVA_HOME%\bin;%JAVA_HOME%\jre\bin;"}
  • 新建{"CLASSPATH":".;%JAVA_HOME%\lib\dt.jar;%JAVA_HOME%\lib\tools.jar;"}
  • 检测 $ java -version, $ java, $ javac

项目中启动前端工作流

在前端项目 package.json 文件所在目录命令行执行 npm install 或 yarn install 安装依赖包即可，HG/Git 版本控制工具中添加忽略目录 node_modules。

安装报错，检查网络环境，建议开全局代理。
使用过程中报错，检查权限和 package.json配置。

单独使用 NPM 包

http-server

不需要使用 Tomcat、Apache HTTP Server、IIS 等大型 Web Server 时，可以选择基于 Node.js 的http-server快速搭建简易 HTTP 服务器。

http-server [path] [options]: 如在当前目录9000端口启动：http-server -p 9000

browser-sync

借助browser-sync编辑代码保存后，浏览器自动即时生效，无需F5刷新。

browser-sync 启动服务，监视各层级 html/css/js 文件，忽略包安装的文件，指定端口：
browser-sync start --server --files "**/*.css, **/*.js, **/*.html" --ignore "**/bower_components, **/node_modules" --port 63342

如本地需要 IntelliJ IDEA 跑 Java 处理相关数据，先启动 browser-sync 占用端口，再启动 IDEA Run/Debug。

一般不会单独使用browser-sync，而是配置在 Gulp 打包工具中启动。

terminal-translate

直接在命令行翻译字符，带空格标点的加英文""包裹

tl keyword

从命令行开启 'spring-boot'

mvn spring-boot:run

Tips:

  • Windows 系统 C 盘留 120G，差不多小容量 SSD 盘大小，所有软件默认安装，去除全家桶幽灵按钮勾选即可。然后配置相关数据目录路径到 D 盘 Temp 目录；
  • npm 安装依赖包可能会因国内网络环境报错，建议采用 yarn 安装。或者开代理。
