node-gyp

npm dependencies

如果一个包引用另一个带有 git URL 的包，则 npm 依赖于预安装的git。

如果 npm 试图安装的包之一是 native 节点模块，并且需要编译 c++ 代码，那么 npm 将使用 node-gyp 来完成该任务。对于 Unix 系统，node-gyp 需要 Python、make 和类似 GCC 的构建链。在 Windows 上，需要 Python 和Microsoft Visual Studio c++。node-gyp 不支持 Python 3。

  • 安装 Python 2.7 配置环境变量
  • 安装 VC++ build Tools

$ npm install -g node-gyp
$ npm install --global --production windows-build-tools # 另一种安装 VC++ build Tools 可选方式，开代理安装
$ npm config set python python2.7
$ npm config set msvs_version 2015

ImageMagick

Download

安装路径不能空格；

选择包含headers.和加入path两项，

设置 ImageMagick 到环境变量:

@echo off
set DFImageMagick=C:\ImageMagick
set PATH=%DFImageMagick%;%PATH%
set CPATH=%DFImageMagick%\include;%CPATH%
set LIBRARY_PATH=%DFImageMagick%\lib;%LIBRARY_PATH%
pause>null
exit

CMake 跨平台编译

Cygwin 是一个用于 Windows 的类 UNIX shell 环境；它由两个组件组成：一个 UNIX API 库，它模拟 UNIX 操作系统提供的许多特性；以及 Bash shell 的改写版本和许多 UNIX 实用程序，它们提供大家熟悉的 UNIX 命令行界面；前一个组件是一个 Windows 动态链接库 (DLL)，后一个组件是一组基于 Cygwin DLL 的程序，其中许多是用未经修改的 UNIX 源代码编译的；它们合在一起提供大家熟悉的 UNIX 环境；

MinGW(Minimalist GNU for Windows)，又称 mingw32，是将 GCC(GNU Compiler C) 编译器和 GNU Binutils 移植到 Win32 平台下的产物，包括一系列头文件(Win32API)、库和可执行文件；另有可用于产生32位及64位Windows可执行文件的MinGW-w64项目，是从原本MinGW产生的分支，如今已经独立发展；

MSYS：Unix-like command line utilities，包括基本的 bash, make, gawk and grep 等等；通常也可以认为是小型的UNIX on Windows；提供在 windows 上模拟 Unix 环境来使用 MinGW；

MSYS 是 GNU 实用程序的集合，如 bash，make，gawk 和 grep，允许构建依赖于传统 UNIX 工具的应用程序和程序。它旨在补充MinGW和cmd shell的缺陷；

MSYS2 是 Windows 的软件发行版和构建平台；其核心是独立重写的 MSYS，基于现代 Cygwin(POSIX兼容层)和 MinGW-w64，旨在与本地 Windows 软件更好地互操作；它提供了一个 bash shell，Autotools，修订控制系统等，用于使用 MinGW-w64 工具链构建本地 Windows 应用程序。
