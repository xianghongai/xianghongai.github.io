Windows 设置

Windows 共享设置

1.Windows+R（运行命令）：secpol.msc

  • 本地策略 - 用户权限分配
    • 拒绝本地登陆：删除其中的Guest
    • 拒绝从网络访问这台计算机：删除其中的Guest
  • 本地策略 - 安全选项
    • 网络访问：本地账户的共享和安全模式：仅来宾-对本地用户进行身份验证，其身份为来宾

2.启用Guest账户

  • Windows 10. 运行命令：compmgmt.msc
    计算机管理 > 系统工具 > 本地用户和组 > 用户 > 右侧Guest鼠标右键 > 属性 > 常规 > 取消勾选“账户已禁用 > 确定
  • Windows 7. 运行命令：control userpasswords
    管理其他账户 > 启用Guest
  • 方法二. 运行命令：secpol.msc
    本地策略 > 安全选项 > 账户：来宾账户状态：已启用 > 确定

3.要共享的文件夹

  • 鼠标右键：属性 > 安全 > 编辑(E) > 添加(D) > 高级(A)... > 立即查找(N) > Guest > 确定
  • 鼠标右键：共享 > 下拉框选择 Guest > 添加(A)

4.上面操作还不能共享

  • 尝试运行命令：Control firewall.cpl
  • 左侧：允许程序或功能通过Windows防火墙 > 勾选：文件和打印机共享

5.查看计算机共享文件

  • 运行命令：compmgmt.msc
    计算机管理 > 系统工具 > 共享文件夹 > 共享

6.关闭密码保护共享

  • 运行命令：control.exe /name Microsoft.NetworkAndSharingCenter
    控制面板 > 网络和Internet > 网络和共享中心 > 更改高级共享设置
    所有网络：密码保护的共享 > 关闭密码保护共享

Windows 7 快捷方式小箭头 通过修改注册表去掉

  1. 启动注册表编辑器(开始->运行->输入regedit)，然后依次展开分支： HKEY_CLASSES_ROOT\lnkfile;
  2. 删除 lnkfile 子项中的 IsShortcut 字符串值项;
  3. 展开分支：HKEY_CLASSES_ROOT\piffile;
  4. 删除 piffile 子项中的 IsShortcut 字符串值项;
  5. 退出注册表编辑器，结束并重启 explorer.exe 进程或者重新启动电脑,即生效。

Windows 10 优化设置

关闭家庭组

services.msc / 控制面板 > 管理工具 > 服务 > HomeGroup Listener 和 HomeGroup Provider 禁用

WindowsDefender

  1. 控制面板 > Windows Defender > 设置 > 实施保护 去掉勾
  2. 服务 WindowsDefenderService 禁用

关闭磁盘碎片整理计划

选中磁盘 > 属性 > 工具 > 对驱动器进行优化和碎片整理 > 优化 > 更改设置 > 取消选择按计划运行

Superfetch 服务

服务 > Superfetch > 启动类型 > 自动（延迟启动）

清理产生的Windows.old文件夹

  1. rd c:windows.old /s
  2. C盘 > 右键 > 属性 > 磁盘清理 > 选中"以前的Windows 安装"复选框 > 确定清理

设置 Windows 10 自动登陆

  1. netplwiz,取消使用计算机必须输入用户名和密码的选项
  2. 双击需要自动登录的账户，输入密码

关闭ipv6

网络共享中心 > 网络连接 > 以太网 > 属性 > 取消ipv6

开启 HybridBoot

  1. powercfg.cpl/控制面板 > 电源选项 > 左侧：选择电源按钮的功能 > 更改当前不可用的设置 > 关机设置 > 勾上启用快速启动
  2. 开启休眠：powercfg–h on，关闭休眠：powercfg –h off

关闭性能特效

sysdm.cpl/系统属性 > 高级 > 性能 > 设置 > 关闭淡出淡入效果

第三方软件服务项

msconfig/系统配置 > 服务 > 勾选：隐藏所有的Microsoft服务 > 去掉不需要运行的服务

关闭计划任务里的隐藏的自启动程序

taskschd.msc/任务计划程序库 > 禁用右侧任务列表不需要的任务

关闭 windows 安全中心服务

services.msc/服务 > Security Center > 禁用

Windows鼠标右键没有“新建”“新建文件夹”选项

方法一，注册表修改

  • Windows+R : 运行regedit
  • 注册表树结点 : HKEY_CLASSES_ROOT\Directory\Background\shellex\ContextMenuHandlers\
  • 左侧树ContextMenuHandlers上右键“新建(N)”-"项(K)"，命名为“New”
  • 点击New，在右侧面板双击**(默认)**
  • 弹出“编辑字符串”对话框，“数值数据(V)”设值为**{D969A300-E7FF-11d0-A93B-00A0C90F2719}**

方法二，运行reg文件

  • 如果没有“新建”
  • Windows+R : 运行notepad
  • 保存下面代码到文本文件，然后重命名为 Restore-Missing-New.reg 运行
Windows Registry Editor Version 5.00
[HKEY_CLASSES_ROOT\Directory\Background\shellex\ContextMenuHandlers\New]
@="{D969A300-E7FF-11d0-A93B-00A0C90F2719}"
  • 如果有“新建”，只是没有“新建文件夹”项
  • Windows+R : 运行notepad
  • 保存下面代码到文本文件，然后重命名为 Restore-Missing-New-Folder.reg 运行
Windows Registry Editor Version 5.00
[HKEY_CLASSES_ROOT\Folder\ShellNew]
"Directory"=""
"IconPath"=hex(2):25,00,53,00,79,00,73,00,74,00,65,00,6d,00,52,00,6f,00,6f,00,\
  74,00,25,00,5c,00,73,00,79,00,73,00,74,00,65,00,6d,00,33,00,32,00,5c,00,73,\
  00,68,00,65,00,6c,00,6c,00,33,00,32,00,2e,00,64,00,6c,00,6c,00,2c,00,33,00,\
  00,00

浏览器

同时运行多个不同配置文件的 Firefox

Windwos+R:firefox -ProfileManager

创建配置文件 - 输入新的配置文件名称，然后一定要选择文件夹，新建文件夹，命名后选择这个新文件夹，不然是随机名 - 完成。

一定要去掉启动时不询问并使用选定的配置文件勾选，再退出，因为要把一个默认配置的默认启动的给其他人用。

复制一份 Firefox 的快捷方式文件，右键属性 - 快捷方式 - 目标 - 加参数--no-remote -p 配置文件名：

完整启动方式为："C:\Program Files\Mozilla Firefox\firefox.exe" --no-remote -p 配置文件名

这样就得到一个属于自已配置的Firefox，把这个快捷方式文件命名为iff之类简短的名字然后放在C:\Windows\System32目录，下次直接Windwos+R:iff就启动这个配置的Firefox了。

同时运行多个不同配置文件的 Chrome

同样复制一个新的Chrome快捷方式文件命名为igc，属性目标中加--user-data-dir="%appdata%\Google\Chrome\配置目录名"，这样得到一个独立的 Google Chrome 实例。

完整启动方式为："C:\Program Files (x86)\Google\Chrome\Application\chrome.exe" --user-data-dir="%appdata%\Google\Chrome\配置目录名"
