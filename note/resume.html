对前端看法

一个团队，需要考虑是否需要 Web 开发，全栈工程师很少，指望其他工种来完成 Web 开发以节省成本，是错误的决策，将会造成项目的失序失控；

一个团队，需要像对待后端技术一样，认真衡量 Web 部分在项目产品中的价值，评估产品 Web 部分的工作量，然后根据这些客观情况，计算前端投入成本；

前端技术，语言和规范很重要，只有熟悉语言之后，才能用好、组织好框架、工具，只有熟悉、尊重和遵循规范之后，才能写出可维护的代码，开发者不能舍本逐末；

前端的难点在数据和状态。
